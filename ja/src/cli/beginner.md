# 初級

Rust の文法を網羅的に学ぶのではなく、バグを一つ修正する過程で必要な知識だけを順番に積み上げます。ゼロからツールを作れるようになるのが理想ですが、そこまでの分量を用意すると最後まで辿り着けません。既存のコードを読んで直す、というスタイルにすることで、最短で「動くものを変えた」体験を積めるようにしています。このガイドを終えると、Rust の基本的な動きを理解しながらバグを修正し、PR を作成して何をどう変えたかを自分の言葉で説明できるようになります。

## 題材のツール

ファイルの行数・文字数・バイト数を表示する CLI ツールを使います。使い方はシンプルで、ファイルを引数に渡すだけです。

`hello.txt` の中身

```text
hello
world
foo
```

ターミナルで次のコマンドを実行します。

```sh
$ cargo run -- hello.txt
```

実行結果

```text
lines: 3  chars: 16  bytes: 16
```

`hello`（5文字）・`world`（5文字）・`foo`（3文字）で13文字、それぞれの行末の改行を加えると16文字です。英語のアスキー文字は1文字が1バイトなので、文字数とバイト数が同じになっています。

## バグの内容

英語のファイルでは正しく動きますが、作者は日本語を想定しておらず、マルチバイト文字でのバグが残っています。

`japanese.txt` の中身

```text
こんにちは
世界
```

日本語ファイルを渡すと chars がバイト数になってしまいます。

```sh
$ cargo run -- japanese.txt
```

実行結果

```text
lines: 2  chars: 23  bytes: 23
```

`こんにちは`（5文字）＋`世界`（2文字）＋改行2文字で計9文字のはずが、23になっています。

期待する出力

```text
lines: 2  chars: 9  bytes: 23
```

条件

- 修正後も既存のテストが通ること
- 日本語ファイルに対するテストを追加すること
