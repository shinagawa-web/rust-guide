# 文字数を正しく数える

前のページで、`chars` がバイト数になってしまう原因が `chars = content.len()` だとわかりました。このページでは、なぜ `len()` が文字数にならないのかを理解したうえで、正しく数えるように直します。

## なぜ len() は文字数にならないのか

`content.len()` が返すのは、文字の個数ではなくバイト数です。文字数とバイト数がずれるのは、文字によって必要なバイト数が違うからです。

Rust の文字列は UTF-8 という方式で文字を表します。UTF-8 では、文字によって 1〜4 バイトを使い分けます。

| 文字 | バイト数 |
|---|---|
| `a` や `1` などの半角英数字 | 1 |
| `é` などのアクセント付きの文字 | 2 |
| `あ` や `世` などの日本語 | 3 |
| `😀` などの絵文字 | 4 |

英語のファイルでは 1 文字がちょうど 1 バイトなので、文字数とバイト数がたまたま一致します。`hello.txt` で `chars` と `bytes` がどちらも `16` だったのはこのためです。

一方、日本語は 1 文字が 3 バイトです。最初に見た `japanese.txt` を例にすると、

```text
こんにちは
世界
```

`こんにちは`（5 文字）と `世界`（2 文字）に行末の改行 2 文字を加えて、文字数は 9 です。しかしバイト数で数えると、日本語 7 文字がそれぞれ 3 バイトで 21 バイト、改行 2 文字が 2 バイトで、合計 23 バイトになります。`len()` はこの 23 を返すので、`chars` に文字数ではなく 23 が入っていたわけです。

## chars().count() で文字数を数える

文字の個数を数えるには、まず中身を 1 文字ずつに区切る必要があります。それを行うのが `chars()` です。

`content.chars()` は中身を 1 文字ずつに分けます。そこに `.count()` をつなげると、その個数、つまり文字数が得られます。前のページで行数を `content.lines().count()` と数えていたのと同じ形です。`lines()` が行ごとに区切ったのに対し、`chars()` は文字ごとに区切る、という違いだけです。

```rust
let chars = content.chars().count();
```

バイトではなく文字の単位で区切ってから数えるので、日本語が混ざっても正しい文字数になります。

## bytes はそのままでよい

直すのは `chars` だけです。`bytes` のほうは、もともとバイト数を表示したい値なので、`content.len()` のままで正しく動いています。

| 値 | 数えたいもの | 使う書き方 |
|---|---|---|
| `chars` | 文字数 | `content.chars().count()` |
| `bytes` | バイト数 | `content.len()` |

`len()` が悪いわけではなく、文字数を数えるのに使っていたのが間違いだった、ということです。

## 修正する

`counter.rs` を開き、`count` の中の `chars` の行を書き換えます。

修正前

```rust
        let lines = content.lines().count();
        let chars = content.len();
        let bytes = content.len();
```

修正後

```rust
        let lines = content.lines().count();
        let chars = content.chars().count();
        let bytes = content.len();
```

変更したのは `chars` の 1 行だけです。`lines` と `bytes` はそのままにします。

## 動かして確認する

直したら、日本語ファイルで確かめます。サンプルリポジトリには日本語のファイルが入っていないので、`sample/japanese.txt` を新しく作ります。

```text
こんにちは
世界
```

`世界` の行のあとにも改行を入れて保存してください。最終行の改行がないと、文字数とバイト数がここで示す値より 1 つずつ少なくなります。

このファイルを渡して実行します。

```sh
$ cargo run -- sample/japanese.txt
```

```text
lines: 2  chars: 9  bytes: 23
```

`chars` が `23` から `9` に変わり、期待していた文字数になりました。`bytes` は `23` のままで、こちらは変わっていません。

英語のファイルも確認しておきます。

```sh
$ cargo run -- sample/hello.txt
```

```text
lines: 3  chars: 16  bytes: 16
```

`hello.txt` の結果は修正前と同じです。英語は 1 文字 1 バイトなので、`chars` の数え方を変えても結果が変わらないことが確認できます。
