# CLI の基本

前のページで Rust の基本構文を押さえました。次のページからはサンプルのツールのコードを読んでいきますが、その前にもう一つ準備しておきます。

このツールはコマンドラインツール（CLI）です。CLI には、普通の関数とは違う「外の世界とのやり取り」があります。具体的には次の 3 つです。

- 引数 ― 起動時に値を受け取る
- 終了コード ― 処理の成否を数値で返す
- 標準出力と標準エラー出力 ― 結果とエラーを別の出口に出す

この 3 つは `main.rs` を読むときに必ず出てきます。先に仕組みを掴んでおきましょう。

## 引数 ― 起動時に値を受け取る

CLI は、起動するときに値を渡せます。このツールで `cargo run -- sample/hello.txt` と打ったときの `sample/hello.txt` が引数です。

Rust では、コマンドライン引数の一覧を `std::env::args()` で受け取ります。これを `.collect()` で `Vec<String>` にまとめると、引数を順番に取り出せます。

```rust
use std::env;

fn main() {
    let args: Vec<String> = env::args().collect();
    println!("0番目: {}", args[0]);
    println!("1番目: {}", args[1]);
}
```

コード冒頭の `use std::env;` は、使う機能をあらかじめ取り込んでおく宣言です。これがあるおかげで `std::env::args()` を `env::args()` と短く書けます。`use` や `::` の詳しい意味は次のページで扱います。

ここで注意したいのが、`args[0]` には引数そのものではなくプログラム自身の名前（実行ファイルのパス）が入る点です。利用者が渡した最初の値は `args[1]` から始まります。

たとえば次のように実行すると、

```sh
$ cargo run -- sample/hello.txt
```

`args[0]` はビルドされた実行ファイルのパス、`args[1]` が `sample/hello.txt` になります。利用者が渡したファイル名は `args[1]` から取り出すことになります。

引数が渡されなかった場合、`args` の要素は `args[0]` の 1 つだけになります。このとき `args[1]` は存在しないので、アクセスすると実行時にプログラムが落ちてしまいます。

## 終了コード ― 処理の成否を数値で返す

CLI は、終わるときに「成功したか失敗したか」を数値で返します。これを終了コード（exit code）と呼びます。

- `0` ― 成功
- `0` 以外 ― 失敗

この数値は、ツールを別のツールやスクリプトから呼び出すときに使われます。呼び出した側は終了コードを見て、処理が成功したかどうかを判断できます。

Rust では、`main` が最後まで普通に終われば終了コードは `0` になります。途中で失敗したことを伝えて終了したい場合は `std::process::exit()` に数値を渡します。

```rust
use std::process;

fn main() {
    println!("処理を中断します");
    process::exit(1);
}
```

終了コードはターミナルで確認できます。直前に実行したコマンドの終了コードは `$?` に入ります。

```sh
$ cargo run -- sample/hello.txt
$ echo $?
0
```

## 標準出力と標準エラー出力 ― 結果とエラーを別の出口に出す

プログラムが文字を出す先には 2 つの系統があります。

- 標準出力（stdout）― 通常の結果を出す側。`println!` が使う
- 標準エラー出力（stderr）― エラーや警告を出す側。`eprintln!` が使う

どちらもターミナルに表示されるので、普段は違いを意識しません。しかし両者は別々の出口になっていて、片方だけをファイルに保存するといった扱いができます。

```sh
$ cargo run -- sample/hello.txt > result.txt
```

`>` は標準出力をファイルに書き出す記法です。この場合、結果（stdout）は `result.txt` に入りますが、エラー（stderr）は画面にそのまま残ります。正常な結果とエラーを混ぜずに分けておけるわけです。

前のページで `eprintln!` を「エラー用」と説明したのは、この標準エラー出力に出すという意味でした。
