# プロジェクトを動かす

issue を解き始める前に、ローカル環境に Rust の実行環境を整えてプロジェクトが動く状態を確認します。

## rustup をインストールする

Rust のインストールには rustup を使います。rustup は Rust のインストーラーで、コンパイラ本体のインストールとバージョン管理を担うツールです。

### Mac

ターミナルで次のコマンドを実行します。

```sh
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

途中で選択肢が出たらそのまま Enter を押して進めます。

### Windows

公式サイトからインストーラーをダウンロードして実行します。

[rustup.rs](https://rustup.rs)

### インストールの確認

インストールが終わったらターミナル（またはコマンドプロンプト）を再起動してください。インストールできたか確認します。

```sh
$ rustc --version
```

バージョン番号が表示されれば成功です。

## cargo とは

rustup をインストールすると、cargo も一緒に入ります。

cargo は Rust のビルドツールです。コードのビルド・テストの実行・外部ライブラリの取得をまとめて行います。Rust での作業は基本的に cargo コマンドを通して進みます。

| コマンド | 役割 |
|---|---|
| `cargo build` | コードをビルドする |
| `cargo run` | ビルドして実行する |
| `cargo test` | テストを実行する |
| `cargo clippy` | コードの問題点を指摘する（リント） |
| `cargo fmt` | コードを整形する（フォーマット） |

## サンプルリポジトリについて

このガイドでは、前のページで紹介した CLI ツールのサンプルリポジトリを使います。Rust を学ぶのに必要な部分だけを取り出した、できる限りシンプルな構成にしています。clone してすぐ動かせます。

## リポジトリを clone する

サンプルリポジトリを clone します。

```sh
$ git clone https://github.com/shinagawa-web/rust-guide-sample-beginner-cli.git
$ cd rust-guide-sample-beginner-cli
```

## 動かしてみる

コードを触り始める前に、手元で動くことを確認しておきます。最初に動作確認しておくことで、後で何か壊れたときに「自分の変更が原因」と確信を持って判断できます。

```sh
$ cargo run -- sample/hello.txt
```

`--` は cargo へのオプションとプログラムへの引数を区切るための記法です。`--` より後ろの `sample/hello.txt` はこのツール自身に渡されます。

初回はビルドが走るので少し時間がかかります。完了すると次のように表示されます。

```text
lines: 3  chars: 16  bytes: 16
```

この出力が確認できれば、ツールは正しく動いています。

## テストを実行する

続いてテストも通ることを確認します。テストが最初から通っていることを把握しておくことで、後でテストが壊れたときに自分の変更が原因だとわかります。

```sh
$ cargo test
```

```text
test result: ok. 1 passed; 0 failed
```

これで準備完了です。
