# 3つの入口の使い分け

どの入口から読み始めるかで、同じ概念でも腑に落ちる早さは変わります。ここでは3つの入口がそれぞれどんな場面で力を発揮するかを並べます。自分に合うものを選ぶ材料にしてください。読み進めるうちに別の入口のほうがしっくりくると気づくこともあります。最初の選択を間違えても困らない、くらいの気持ちで眺めてください。

## レベル別

[レベル別](../by-level/README.md)は、内容を初級・中級・上級という難易度で整理した入口です。ここで大切にしているのは、自分の現場や目的に必要なレベルを選ぶという考え方です。初級・中級・上級は上から下への一本のはしごではなく、それぞれが別の必要に応える棚のようなものだと思ってください。

たとえば社内ツールを書きたいだけの人と、ライブラリの実装に踏み込みたい人とでは、必要になる知識の深さがそもそも違います。前者にとっては初級の範囲で十分なことも多く、無理に上級まで触れる必要はありません。逆に、込み入った所有権の話や非同期の内部に関心がある人は、最初から中級・上級を覗いてみてもかまいません。今の自分に必要な深さはどこかという視点で選ぶのが、この入口の使い方です。

## 言語別

[言語別](../by-language/README.md)は、Go・Python・TypeScript・C・C++・Java など、これまで使ってきた言語に置き換えながら Rust を基礎から説明する入口です。同じ仕組みでも、馴染みのある言語のどれに当たるか、どこが違うかという形で示されると、抽象的なまま受け取るより理解が進みます。

この入口でも、基礎は基礎として一通り説明します。違うのは説明の切り口で、Rust の仕組みを、あなたがすでに知っている言語のどの概念に当たるかという形で示します。馴染みのある部分は置き換えてすっと読めるので、その分 Rust ならではのところに時間を使えます。すでに持っている言語の地図の上に Rust を重ねて読みたい人に向いた入口です。

## 作りたいもの別

頭の中にすでに作りたいものがある人に向いた入口です。[Web バックエンド](../web-backend/README.md)・[CLI ツール](../cli/README.md)・[WebAssembly](../wasm/README.md)・[低レイヤ・システム](../low-level/README.md)・[組み込み](../embedded/README.md)といった目的ごとに独立したガイドがあり、その目的を達するために何を知っておくとよいかという順序で並んでいます。

この入口が効くのは、学ぶ動機が「Rust そのもの」ではなく「これを作りたい」のほうにある場合です。手を動かして動くものができていく手応えが学習を引っ張ってくれますし、目的に直接関わる概念から触れるので、なぜその仕組みが必要なのかも実感しやすくなります。逆に、まだ何を作るか決まっていない段階では選びにくい入口でもあります。そのときは上の2つを先に見てみてください。

## 迷ったときの指針

どれにするか決めかねたら、次のように考えてみるのはどうでしょうか。今の自分にどこまでの深さが必要かをはかりにかけたいなら、レベル別が向きます。すでに知っている言語の上に Rust を重ねて読みたいなら、言語別が取りかかりやすいでしょう。作りたいものがはっきりしているなら、作りたいもの別から入るのが素直です。これはあくまで目安で、当てはまらなければ気にせず別の入口を選んでください。

## 入口は行き来できる

ひとつ覚えておいてほしいのは、入口を選んだらそこに縛られるわけではない、ということです。作りたいもの別で読み進めていて所有権の理解が足りないと感じたらレベル別を、説明がうまく像を結ばないと感じたら言語別を、というように、途中で別の入口へ移ったり、複数を並行して使ったりして問題ありません。3つは別々の本ではなく、同じ内容への別々の入口です。自分にとっていちばん読みやすい道筋を、そのつど選び直しながら進んでください。
