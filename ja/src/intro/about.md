# このガイドについて

**Why-first な Rust 学習ガイド**です。「どうやって書くか」より先に「**なぜそうなっているのか**」を起点に、Rust を学べることを目指しています。

ライフタイムの記号の書き方を覚える前に borrow チェッカーが何を防ごうとしているかを理解する——そんな順序で読めるように構成しています。

## 想定読者

何らかのプログラミング経験があり、これから Rust を学ぶ人を想定しています。構文をゼロから手取り足取り教えるより、「既存の知識との差分」や「設計の理由」に踏み込む書き方を優先しています。

なぜ既存のドキュメントと別にこのガイドを作ったのか、その問題意識は [なぜこのガイドを作ったか](why-this-guide.md) にまとめています。

## 3つの入口

同じ概念でも、読者のバックグラウンドや目的によって「最初に学ぶべき順序」は違います。このガイドは1本道のロードマップを押し付けず、自分に合った入口から読み始められるよう、コンテンツを3つの軸で整理しています。

| 入口 | こんな人に |
|---|---|
| [**作りたいもの別**](../by-motivation/README.md) | CLI ツール、Web バックエンド、WebAssembly など、目的が先にある人 |
| [**レベル別**](../by-level/README.md) | 初級・中級・上級と、段階を追って積み上げたい人 |
| [**言語別**](../by-language/README.md) | Go・Python・TypeScript など、使ってきた言語に置き換えて説明されると理解が進む人 |

どの入口を選ぶか迷ったら、[3つの入口の使い分け](how-to-read.md) を参照してください。

## コードを動かしながら読む

コード例には極力 [Rust Playground](https://play.rust-lang.org/) へのリンクを付けています。ローカル環境を用意しなくても、ブラウザ上でそのまま実行して確認できます。詳しくは [Rust Playground の使い方](playground.md) を参照してください。
