# なぜこのガイドを作ったか

Rust の入門ドキュメントはすでに数多くあります。公式の The Book をはじめ、丁寧に書かれた資料が揃っています。それでも別のガイドを作ったのは、私たち自身が既存のドキュメントで Rust を学ぼうとしたときに、いくつか共通した引っかかりを感じたからです。

ここでは、その引っかかりを5つの課題として書き出し、それぞれに対して本ガイドがどう応えているかを示します。応答の細かい中身はリンク先のページに譲りますが、まずは「なぜこういう作りになっているのか」を理解してから読み進めてもらえると、このガイド全体の意図が掴みやすくなるはずです。

## 「なぜ」から理解する

多くのドキュメントは、構文や書き方の説明はとても丁寧です。けれども、その仕組みがそもそも何のために言語に備わっているのか、という説明は薄くなりがちでした。たとえばライフタイムの記号 `'a` を、いつどこに書くかというルールから先に教わると、記号そのものが目的のように見えてしまいます。本当は、コンパイラが参照の寿命について何を心配しているのかを先に知っていれば、記号はその心配を伝えるための道具として自然に読めます。

このガイドは、仕組みの理由から入る Why-first の方針を貫いています。考え方の全体像は [このガイドについて](about.md) を参照してください。

## 自分に合った入口から始める

CLI ツールを作りたい人も、ブラウザで動かす WebAssembly に興味がある人も、低レイヤーの挙動を知りたい人も、既存のドキュメントではたいてい同じ章を同じ順序でたどることになります。けれども、何を最初に学ぶと腑に落ちやすいかは、その人の目的や背景によって変わります。一本道のロードマップは、誰かにとっては遠回りになってしまいます。

そこでこのガイドは、目的・背景に応じた複数の入口を用意し、自分に合ったところから読み始められるようにしました。入口の一覧は [このガイドについて](about.md) に、選び方に迷ったときの指針は [3つの入口の使い分け](how-to-read.md) にまとめています。

## 必要なレベルを選んで学ぶ

Rust は覚えることが多い言語です。所有権、ライフタイム、トレイト、スマートポインタ、unsafe と、概念が次々に出てきます。困るのは、そのうち自分の目的にはどこまで必要なのかを判断する材料が、ドキュメント側からはなかなか得られないことでした。結果として、全部やらなければいけない気持ちになり、手が止まってしまいます。

必要な深さは、人によっても、現場によっても、作るものによっても違います。このガイドはレベルで区分けすることで、自分に必要なレベルを選んで読めるようにしています。詳しくは [レベル別](../by-level/README.md) を参照してください。

## 使ってきた言語を足場にする

プログラミング経験がある人なら、すでに何らかの言語で考え方の土台ができています。ところが既存のドキュメントは、その土台に置き換えて Rust を説明してはくれません。ゼロから Rust の語彙だけで積み上げる説明が多く、せっかく持っている足場を使えないまま読み進めることになります。

このガイドは、あなたがこれまで使ってきた言語に置き換えながら、Rust を基礎から説明します。言語ごとの入口は [言語別](../by-language/README.md) にあります。

## 最初から日本語で書く

日本語で読める資料の多くは、英語で書かれたものを訳したものです。原文に忠実であろうとするほど英語の構文がそのまま残りやすく、直訳調の文体になります。Rust はただでさえ概念が難しいので、そこに読みにくさが重なると、理解の負荷が二重になってしまいます。

だからこのガイドは、訳すのではなく、最初から日本語で書いています。日本語で考えて日本語で説明することで、概念の難しさだけに集中して向き合えるようにすることを目指しました。
