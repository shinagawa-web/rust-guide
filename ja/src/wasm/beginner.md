# 初級

Rust の文法を網羅的に学ぶのではなく、既存の小さなプロジェクトを読み、バグを一つ修正する過程で必要な知識だけを順番に積み上げます。ゼロからツールを作れるようになるのが理想ですが、そこまでの分量を用意すると最後まで辿り着けません。既存のコードを読んで直す、というスタイルにすることで、最短で「動くものを変えた」体験を積めるようにしています。このガイドを終えると、Rust で書いた処理がブラウザの中で動き JavaScript から呼ばれる仕組みを理解しながらバグを修正し、何をどう変えたかを自分の言葉で説明できるようになります。

主役はツールそのものより、Rust と JavaScript が受け渡しをする「境界」です。読み終えたときに残ってほしいのは、Rust で書いた関数がブラウザの中で動き、JavaScript から呼ばれている、という手応えです。

## 題材のツール

2つのテキストを比べて、どこが変わったかを表示する差分ツールを使います。ブラウザで開いてテキストを2つ入れると、消えた部分と増えた部分が色分けで表示されます。違いを数えているのは Rust で書いた関数で、それをブラウザの中で動かしています。

たとえば次の2つを比べます。

テキストA

```text
I have a cat.
```

テキストB

```text
I have a dog.
```

差分の表示

```text
I have a [-cat-][+dog+].
```

`[-cat-]` が消えた部分、`[+dog+]` が増えた部分です。`cat` が `dog` に変わったことが読み取れます。

## バグの内容

英語のテキストでは正しく動きますが、日本語のように1文字が複数バイトになる文字を入れると、差分が文字の途中で切れて壊れます。

次の2つを比べます。

テキストA

```text
吾輩は猫である。
```

テキストB

```text
吾輩は犬である。
```

「猫」が「犬」に変わっただけなので、差分もその部分だけを指してほしいところですが、文字の途中で区切られて文字化けした差分が出てしまいます。

```text
吾輩は[-�-][+�+]�である。
```

文字ではなくバイトの並びとして比べているために、1文字の途中で区切ってしまうのが原因です。

期待する表示

```text
吾輩は[-猫-][+犬+]である。
```

条件

- 修正後も既存のテストが通ること
- 日本語のテキストに対するテストを追加すること

## このガイドの進め方

全 8 ページ。コードを手元で動かしながら読み進めると、約 2 時間で完走できます。

以下の順で進みます。

1. WebAssembly とは / なぜ Rust

7. テストを追加する
