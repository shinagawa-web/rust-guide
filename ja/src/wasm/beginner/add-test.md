# テストを追加する

前のページで、`diff` を文字単位に直して、日本語でも正しく差分が取れるようになりました。ただ、このまま終えると、あとでコードを変えたときに日本語がまた崩れても気づけません。日本語が正しく差分になることを確かめるテストを足して、崩れたらすぐ気づけるようにします。

`src/lib.rs` の末尾には、もともとテストが書かれています。まずそれを読み、書き方をつかんでから、日本語のテストを足します。

## 既存のテストを読む

`src/lib.rs` の末尾はこうなっています。

```rust
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn diffs_english_words() {
        assert_eq!(diff("I have a cat.", "I have a dog."), "I have a [-cat-][+dog+].");
    }

    #[test]
    fn no_change_returns_same_text() {
        assert_eq!(diff("hello", "hello"), "hello");
    }

    #[test]
    fn pure_insertion() {
        assert_eq!(diff("ac", "abc"), "a[+b+]c");
    }

    #[test]
    fn pure_deletion() {
        assert_eq!(diff("abc", "ac"), "a[-b-]c");
    }
}
```

初めて見る書き方が並んでいるので、上から順に見ていきます。

```rust
#[cfg(test)]
mod tests {
```

`mod tests` は、テストをまとめておくまとまりです。`mod` はいくつかのものを1つにまとめる書き方です。直前の `#[cfg(test)]` は、このまとまりをテストのときだけ有効にするための目印です。`cargo test` で実行するときだけコンパイルされ、`wasm-pack build` では無視されます。

```rust
    use super::*;
```

`super` は、`mod tests` を囲んでいるまとまり、つまり `lib.rs` の本体を指します。`use super::*;` で、本体にある `diff` を、テストの中からそのままの名前で使えるようにしています。これがなければ、`diff` を呼ぶたびに `super::diff` と書くことになります。

```rust
    #[test]
    fn diffs_english_words() {
```

`#[test]` は「これはテスト関数です」という目印です。この目印が付いた関数が `cargo test` で実行されます。`diffs_english_words` は関数の名前で、何を確かめるテストかがわかるように付けられています。

```rust
        assert_eq!(diff("I have a cat.", "I have a dog."), "I have a [-cat-][+dog+].");
```

`assert_eq!` は、2つの値が等しいことを確かめる書き方です。左に実際の値、右に期待する値、という順で書くのが慣習です。等しければそのまま通り、違っていればその場でテストが失敗します。ここでは、`I have a cat.` と `I have a dog.` を `diff` にかけた結果が、`I have a [-cat-][+dog+].` になるはずだと確かめています。

残りの3つも同じ形で、別の場合を確かめています。同じ文章どうしなら変化なしでそのまま返ること（`no_change_returns_same_text`）、片方にだけ文字が増えたとき（`pure_insertion`）、片方から文字が消えたとき（`pure_deletion`）です。

## 日本語のテストを追加する

既存のテストにならって、日本語でも正しく差分になることを確かめるテストを足します。`mod tests` の中、既存の4つのテストに続けて次を書き加えます。

```rust
    #[test]
    fn diffs_japanese() {
        assert_eq!(diff("吾輩は猫である。", "吾輩は犬である。"), "吾輩は[-猫-][+犬+]である。");
    }
```

形は既存のテストと同じで、渡す文章を日本語にしただけです。期待する値の `吾輩は[-猫-][+犬+]である。` は、前のページで直したあとの正しい表示です。「猫」から「犬」への変化だけが差分になっていれば、日本語が正しく差分になっていると確かめられます。

書き加えると、`mod tests` の中にテスト関数が5つ並びます。

## テストを実行する

テストは `cargo test` で実行します。

```sh
$ cargo test
```

```text
test result: ok. 5 passed; 0 failed
```

5つのテストがどれも通りました。最初からあった英語の4つと、いま足した日本語の1つです。前のページの修正が、英語のテストを壊さずに日本語にも対応できていることが確かめられました。

## 直していなければ失敗する

もし前のページの修正をしていなかったら、いま足したテストは通りません。バイト単位のままだと結果が `吾輩は�[-��-][+��+]である。` になり、期待した表示と食い違って失敗します。

```text
assertion `left == right` failed
  left: "吾輩は�[-��-][+��+]である。"
 right: "吾輩は[-猫-][+犬+]である。"
```

このテストがあれば、あとからの変更で `diff` がまたバイト単位に戻ってしまっても、`cargo test` で失敗としてすぐ気づけます。直した内容が元に戻っていないかを確かめられる、これがテストを足す理由です。

これで、最初に決めた2つの条件、既存のテストが通ることと日本語のテストを追加することの両方を満たせました。次のページでは、ここまでの変更をふり返り、何をどう変えたかをまとめます。
