# プロジェクトを動かす

コードを読み始める前に、サンプルのプロジェクトを手元で動かして、ブラウザに差分ツールが表示される状態を作ります。最初に動くことを確かめておくと、あとで動かなくなったときに、自分の変更が原因だと確信を持って切り分けられます。

この初級編のコードはブラウザの中で動きます。そのため、ふつうの Rust プログラムを動かすときとは道具立てが少し違います。書いた Rust をブラウザが読める形（wasm）に変換し、それをブラウザに読み込ませる、というひと手間が要ります。

## rustup で Rust を入れる

Rust のインストールには rustup を使います。rustup は Rust のインストーラーで、コンパイラ本体と、ビルドツールの cargo が一緒に入ります。

### Mac・Linux

ターミナルで次のコマンドを実行します。

```sh
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

途中で選択肢が出たらそのまま Enter を押して進めます。

### Windows

公式サイトからインストーラーをダウンロードして実行します。

[rustup.rs](https://rustup.rs)

### インストールの確認

インストールが終わったらターミナル（Windows はコマンドプロンプトなど）を再起動します。Mac・Linux では、再起動の代わりに `source "$HOME/.cargo/env"` を実行すると、今のシェルにそのまま反映できます。

入ったか確認します。

```sh
$ rustc --version
```

バージョン番号が表示されれば成功です。

## wasm-pack を入れる

次に wasm-pack を入れます。これがこの初級編の要になる道具で、2つの仕事をします。1つは、書いた Rust をブラウザが読める wasm に変換すること。もう1つは、その wasm をブラウザに読み込ませ、中の関数を JavaScript から呼べるようにするための、橋渡し役の JavaScript を自動で用意することです。

さきほど入った cargo を使ってインストールします。

```sh
$ cargo install wasm-pack
```

入ったか確認します。

```sh
$ wasm-pack --version
```

バージョン番号が表示されれば成功です。

## リポジトリを clone する

サンプルリポジトリを clone します。題材として必要な部分だけを取り出した、できる限りシンプルな構成にしています。

```sh
$ git clone https://github.com/shinagawa-web/rust-guide-sample-beginner-wasm.git
$ cd rust-guide-sample-beginner-wasm
```

## wasm にビルドする

wasm-pack でビルドします。

```sh
$ wasm-pack build --target web
```

`--target web` は、でき上がった wasm をブラウザから直接読み込む形にする指定です。

初回は必要なもののダウンロードが走るので少し時間がかかります。完了すると `pkg/` というフォルダができ、中に2つの主役が入ります。

```text
pkg/
├── textdiff_bg.wasm   ← Rust から変換された wasm 本体
└── textdiff.js        ← それをブラウザに読み込ませる橋渡しの JavaScript
```

`textdiff.js` を通して読み込むと、Rust で書いた `diff` 関数がブラウザの中から呼べるようになります。この橋渡しの中身は4章で読みます。

## ブラウザで開く

でき上がったものをブラウザで開きます。ここで1つ注意があります。このページは JavaScript のモジュールとして読み込まれ、その中で wasm を取ってきます。ブラウザは、`file://` で直接開いたページにはこの読み込みを安全のために許していないので、ローカルサーバーを立てて `http://` で配る必要があります。

Python が入っていれば、追加のインストールなしにサーバーを立てられます。プロジェクトのフォルダで実行します。

```sh
$ python3 -m http.server 8000
```

この状態でブラウザから <http://localhost:8000/> を開くと、差分ツールが表示されます。

開いた時点では、既定で日本語の例が入っていて、差分が文字化けした形で表示されます。これはトップページで見た、日本語で差分が崩れる例そのものです。後で直すので、いまはそのままで構いません。

ツール自体がちゃんと動いていることは、テキストA・テキストB を英語の文章に入れ替えて確かめられます。変えた部分が `[-消えた部分-]` と `[+増えた部分+]` の色分けで表示されれば、Rust で書いた関数がブラウザの中で動いています。

止めるときはターミナルで Ctrl-C を押します。

## テストも通しておく

差分を計算する部分は、ブラウザに持っていかなくても、ふつうのパソコン上でそのままテストできます。

```sh
$ cargo test
```

```text
test result: ok. 4 passed; 0 failed
```

最初にテストが通ることを確かめておくと、あとでテストが落ちたときに、自分の変更が原因だとわかります。

これで準備は完了です。次はコードを読む前に、題材に出てくる Rust の文法をひととおり見ておきます。
