# 差分を正しく取る

`diff` は `build_diff` と `render` の2段で動き、その全体がバイトの単位で進みます。英語のような半角の文字ならこれで正しく差分が取れますが、日本語を入れると表示が崩れます。`吾輩は猫である。` と `吾輩は犬である。` を比べると、次のように出ます。

```text
吾輩は�[-��-][+��+]である。
```

「猫」が「犬」に変わっただけなので、本当はその部分だけを差分にしてほしいところです。ここでは、なぜバイト単位だと崩れるのかを押さえたうえで、文字単位で比べるように直します。

## なぜバイト単位だと崩れるのか

Rust の文字列は UTF-8 という方式で文字を表します。UTF-8 では、文字によって使うバイト数が違います。

| 文字 | バイト数 |
|---|---|
| `a` や `1` などの半角英数字 | 1 |
| `é` などのアクセント付きの文字 | 2 |
| `あ` や `猫` などの日本語 | 3 |
| `😀` などの絵文字 | 4 |

英語のような半角の文字は1文字が1バイトなので、バイトの区切りがそのまま文字の区切りになります。バイト単位で比べても、文字の途中で切れることはありません。`cat` と `dog` がきれいに差分になったのはこのためです。

日本語は1文字が3バイトです。ここで問題になるのは、別々の文字でもバイトの一部がたまたま重なることがある、という点です。`猫` と `犬` は、3バイトのうち先頭の1バイトが同じです。これをバイト単位で比べると、その先頭の1バイトだけが「共通」と判定され、残りの2バイトずつが「消えた」「増えた」の断片になります。

`render` は、その断片のバイトを集めて文字列に戻そうとします。ですが、文字の一部だけを取り出したバイトは、それだけでは文字として成り立ちません。文字に戻せなかったところが `�` になり、`吾輩は�[-��-][+��+]である。` という崩れた表示になっていたわけです。

## 文字単位で比べる

崩れる原因は、文字列をバイトの並びとして扱っていたことでした。そこで、バイトではなく文字の並びとして扱うように変えます。`chars()` は、文字列を1文字ずつに区切るものです。これを使って先に文字の並びにしておけば、`build_diff` が比べる単位も、`render` が集める単位も1文字になり、文字の途中で切れることがなくなります。

変えるのは、コードの中で「バイト」を指していた4か所です。

## 直す

### 1. diff ― 文字の並びに変えてから渡す

`diff` の中で、文字列をバイトの並び（`as_bytes()`）にして渡していたのを、文字の並び（`chars()`）にします。

修正前

```rust
pub fn diff(a: &str, b: &str) -> String {
    let ops = build_diff(a.as_bytes(), b.as_bytes());
    render(&ops)
}
```

修正後

```rust
pub fn diff(a: &str, b: &str) -> String {
    let a: Vec<char> = a.chars().collect();
    let b: Vec<char> = b.chars().collect();
    let ops = build_diff(&a, &b);
    render(&ops)
}
```

`a.chars().collect()` は、文字列を1文字ずつに区切って `Vec<char>`、つまり文字の並びにします。それを `build_diff` に渡します。

### 2. Op ― 1バイトではなく1文字を持つ

差分の1要素が持つ中身を、1バイト（`u8`）から1文字（`char`）にします。

修正前

```rust
struct Op {
    kind: Kind,
    byte: u8,
}
```

修正後

```rust
struct Op {
    kind: Kind,
    ch: char,
}
```

### 3. build_diff ― 文字の並びを受け取る

`build_diff` が受け取る型を、バイトの並び（`&[u8]`）から文字の並び（`&[char]`）にします。

修正前

```rust
fn build_diff(a: &[u8], b: &[u8]) -> Vec<Op> {
```

修正後

```rust
fn build_diff(a: &[char], b: &[char]) -> Vec<Op> {
```

あわせて、関数の中で `Op` を作っている5か所（`ops.push(Op { ... })`）の `byte:` を `ch:` に書き換えます。`a[i] == b[j]` のように中身を比べているところは、そのままで動きます。文字どうしもそのまま `==` で比べられるからです。

### 4. render ― 文字をそのままつなぐ

`render` は、集めた中身を文字列に戻すのに `String::from_utf8_lossy` を使っていました。中身が1文字になったので、文字をそのまま `String` に追加するだけで済みます。

修正前

```rust
        let mut bytes = Vec::new();
        while i < ops.len() && &ops[i].kind == kind {
            bytes.push(ops[i].byte);
            i += 1;
        }
        let text = String::from_utf8_lossy(&bytes);
```

修正後

```rust
        let mut text = String::new();
        while i < ops.len() && &ops[i].kind == kind {
            text.push(ops[i].ch);
            i += 1;
        }
```

断片を文字に戻す処理そのものが要らなくなり、`from_utf8_lossy` の行は消えます。囲み記号を付ける `match` の部分は変えません。

## 動かして確認する

Rust のコードを変えたので、wasm を作り直します。

```sh
$ wasm-pack build --target web
```

でき上がったら、ブラウザで開いているページを再読み込みします。ローカルサーバーを止めていた場合は、もう一度 `python3 -m http.server 8000` で立ててから開きます。

既定で入っている日本語の例が、正しい差分になります。

```text
吾輩は[-猫-][+犬+]である。
```

「猫」が「犬」に変わった、その1文字だけが差分として表示されました。英語の文章も、これまでどおり正しく表示されます。

```text
I have a [-cat-][+dog+].
```

計算する部分だけは、ブラウザに持っていかなくてもテストできます。既存のテストが通ることも確かめておきます。

```sh
$ cargo test
```

```text
test result: ok. 4 passed; 0 failed
```

英語のテストは修正前と同じように通ります。文字単位に変えても、英語の結果は変わらないからです。次のページでは、日本語が正しく差分になることを確かめるテストを追加します。
