# 中級をふりかえる

中級では、初級で仕上げた差分ツールを出発点に、いくつもの章を重ねて手を入れてきました。この章で、その差分ツールがどこまで来たのかを振り返り、この先どんなふうに手を加えていけるかを見ておきます。

初級のときは、ブラウザの中だけで動く、テキストを比べて差分を返すだけの Rust でした。それが、世界のどこからでも呼び出せる、公開された HTTP エンドポイントになっています。同じ差分の計算が、ブラウザの中から、自分で公開したエッジへと場所を移しました。

ここまで手を入れてきたコードの到達点は、サンプルリポの [`json-api` ブランチ](https://github.com/shinagawa-web/rust-guide-sample-intermediate-wasm/tree/json-api) にまとまっています。うまく動かないときや、どこで間違えたのか分からなくなったときは、このコードと見比べてみてください。写し間違いのような小さなずれは自分では気づきにくいので、差を見れば原因の手がかりになります。

## できるようになったこと

いまの差分ツールは、これだけのことができます。

- 初級で手を入れた差分の核を、同じ Rust のまま、ブラウザの外のエッジで動かせる
- POST で送られてきた JSON の2つのテキストを受け取り、差分を JSON で返す
- POST 以外で呼ばれたら 405、JSON にならない本文には 400 と、失敗の理由を番号で返す
- ブラウザの HTML からも呼び出せるよう、許可のヘッダを付けてある
- `wrangler deploy` で、世界のどこからでも呼び出せる公開の URL に載せられる

ブラウザの中で差分を計算するだけだったものが、誰かに呼んでもらえる一つのサービスになりました。

## 身についた考え方

作る過程で、いくつもの考え方に触れました。文法として先に覚えたものではなく、差分ツールに必要になったから一つずつ使ってきたものです。

- 同じ Rust を、ブラウザの外でも動かせる。wasm に変換すれば、エッジのランタイム（V8 アイソレート）の上で動く
- エッジで動くプログラムは、リクエストを受けてレスポンスを返す、という一つの形をしている（fetch ハンドラ）
- 構造のあるデータは、serde に任せて JSON と `struct` を行き来させる。`derive` を付けるだけで、変換のコードは自分で書かなくていい
- 失敗は `Result` で表し、呼ぶ側に意味のある形で返す。読み取りの失敗を、そのままのエラーではなく `400` という番号に翻訳する
- `?` は、失敗したら中断する印としても、成功した値を取り出す道具としても使える
- ブラウザには別のドメインへの呼び出しを止める仕組みがあり、呼ばれる側が許可のヘッダを返すことで応える
- 手元で `wrangler dev` を回しながら手を入れ、動きを確かめてから `wrangler deploy` で公開する

どれも、「送られてきたテキストを受け取りたい」「失敗の理由を伝えたい」「ブラウザから呼びたい」といった目的が先にあって、それを叶える道具として出てきました。目的から入ったぶん、次に別のものをエッジで動かすときも、同じ道具を自分の判断で持ち出せるはずです。

## この先やってみたいこと

差分ツールは「完成」というより、ここから好きに機能を足していける土台ができた、というほうが近いです。まずは、この差分ツールそのものに手を加える方向があります。

- ブラウザのクライアントを作り込む。いまは最小の HTML ですが、差分を記号ではなく色で見せたり、入力欄を使いやすくしたりできます
- 差分の単位を変える。いまは文字ごとに比べていますが、単語や行ごとに差分を取る形にもできます
- 独自のドメインで公開する。`workers.dev` の URL の代わりに、自分の持っているドメインを割り当てられます
- 大きすぎる入力を断る。送られてくるテキストにサイズの上限を決めて、超えたら受け付けないようにできます

差分ツールの外に目を向けると、同じ形で作れる、役に立つものがいろいろあります。エッジは世界中の利用者の近くでいつでも動くので、こんなものと相性がいいです。

- 短縮 URL のサービス。長い URL を短くして、開かれたら本来のページへ飛ばす。どこから開かれても、近くの拠点がすぐに転送します
- サイトに付ける小さな機能。お問い合わせの受け口や、記事の「いいね」を数える口など。専用のサーバーを持たずに、アクセスが増えても最寄りで応えます
- 文章を整えて返す道具。送られたテキストの表記ゆれや余分な空白を直して返す、差分ツールと同じ作りのものです。使う人の近くで動くので、どこから呼んでも速く応えます

前半は、中級でやってきたやり方、つまり手元で `wrangler dev` を回して確かめながら手を入れるやり方で、そのまま足していけます。後半は、エッジで何が作れるのかを、もう一段広げる入り口です。まずは、自分が「あったら便利」と思うものから試してみてください。それがいちばん、身につく近道です。
