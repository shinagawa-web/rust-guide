# diff をエッジから呼ぶ

1章で、リクエストを受けてレスポンスを返す受け皿（`fetch` ハンドラ）を動かしました。2章で、その裏でリクエストが JavaScript を通って wasm の Rust に届く道すじを見ました。土台はもう整っています。

この章でやるのは、その受け皿に初級の差分コードを差し込むことです。いまは決まった文字列を返すだけのハンドラを、差分を計算して返すハンドラに変えます。ただし、比べる2つのテキストは、この章ではコードに直接書いておきます。送られてきたテキストを読み取るのは、次の章の仕事です。まずは、初級の diff がエッジの上で動くことを目で確かめます。

## ハンドラから diff を呼ぶ

開くのは、1章でも見た `src/lib.rs` です。いまはこうなっています。

```rust
#![allow(dead_code)]

use worker::*;

mod diff;

#[event(fetch)]
async fn fetch(_req: Request, _env: Env, _ctx: Context) -> Result<Response> {
    Response::ok("textdiff worker is running")
}
```

`Response::ok(...)` で決まった文字列を返しているだけの状態です。ここを、差分を計算して返すように書き換えます。

```rust
use worker::*;

mod diff;

#[event(fetch)]
async fn fetch(_req: Request, _env: Env, _ctx: Context) -> Result<Response> {
    let a = "I have a cat.";
    let b = "I have a dog.";
    Response::ok(diff::diff(a, b))
}
```

変えたのは、ハンドラの中身と、先頭の1行です。

比べる2つのテキストを `a` と `b` に置き、`diff::diff(a, b)` で差分を計算しています。`diff::diff` は、`mod diff;` で取り込んだ `src/diff.rs` の中の関数です。初級で文字化けを直した、あの文字ごとに差分を取るコードそのものです。返ってくるのは `String` で、消えた部分を `[-cat-]`、増えた部分を `[+dog+]` のように印で囲んだ文字列です。それをそのまま `Response::ok(...)` に渡して、レスポンスとして返します。

`a` と `b` をコードに直接書いているのは、この章ではまだリクエストの中身を読まないからです。送られてきたテキストを取り出すには、リクエストの本文を解釈するもうひと手間が要ります。それは次の章でやります。ここでは、差分のコードがエッジの上でちゃんと動くことだけに集中します。

先頭から消えた `#![allow(dead_code)]` は、1章で「まだ使っていないコードへの警告を黙らせておくもの」と書いた1行です。いま `diff::diff` を呼んだことで、`mod diff;` で取り込んだコードは実際に使われるようになりました。警告は出なくなるので、黙らせるこの1行はもう要りません。役目を終えたので外します。
