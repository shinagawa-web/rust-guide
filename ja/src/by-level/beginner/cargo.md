# プロジェクトを動かす

issue を解き始める前に、ローカル環境に Rust の実行環境を整えてプロジェクトが動く状態を確認します。

## rustup をインストールする

Rust のインストールには rustup を使います。rustup は Rust のインストーラーで、コンパイラ本体のインストールとバージョン管理を担うツールです。

```sh
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

途中で選択肢が出たらそのまま Enter を押して進めます。インストールが終わったらシェルを再起動してください。

確認します。

```sh
rustc --version
```

バージョン番号が表示されれば成功です。

## cargo とは

rustup をインストールすると、cargo も一緒に入ります。

cargo は Rust のビルドツールです。コードのビルド・テストの実行・外部ライブラリの取得をまとめて行います。Rust での作業は基本的に cargo コマンドを通して進みます。

| コマンド | 役割 |
|---|---|
| `cargo build` | コードをビルドする |
| `cargo run` | ビルドして実行する |
| `cargo test` | テストを実行する |
| `cargo clippy` | コードの問題点を指摘する（リント） |
| `cargo fmt` | コードを整形する（フォーマット） |

## サンプルリポジトリについて

このガイドでは、現場の Rust プロジェクトを模したサンプルリポジトリを使います。実際の現場ではもっと複雑ですが、このガイドに必要な部分だけを取り出した、できる限りシンプルな構成にしています。

中身は axum で書かれた Web API で、ユーザー情報を返す 2 つのエンドポイントがあります。

| エンドポイント | 説明 |
|---|---|
| `GET /users` | ユーザー一覧を返す |
| `GET /users/:id` | 指定した ID のユーザーを返す |

データは DB には保存されておらず、コード内にハードコードされています。DB のセットアップはこのガイドの学習範囲外であり、関係のない概念で混乱しないよう省いています。clone してすぐ動かせます。

## リポジトリを clone する

サンプルリポジトリを clone します。

```sh
git clone https://github.com/shinagawa-web/rust-guide-sample-beginner-api.git
cd rust-guide-sample-beginner-api
```

## サーバーを起動する

コードを触り始める前に、手元で動くことを確認しておきます。最初に動作確認しておくことで、後で何か壊れたときに「自分の変更が原因」と確信を持って判断できます。

```sh
cargo run
```

初回は依存ライブラリのダウンロードとビルドが走るので数分かかります。`Listening on http://localhost:3000` と表示されたら起動完了です。

別のターミナルを開いてアクセスしてみます。一覧取得と個別取得、2 本のエンドポイントがあるので両方確認します。

```sh
curl http://localhost:3000/users
curl http://localhost:3000/users/1
```

JSON が返ってくれば動いています。確認できたら `Ctrl+C` でサーバーを止めます。

## テストを実行する

サーバーを止めたら、続いてテストも通ることを確認します。テストが最初から通っていることを把握しておくことで、後でテストが壊れたときに自分の変更が原因だとわかります。

テストは 3 つあります。一覧取得・個別取得・存在しない ID へのアクセス（404）をそれぞれ確認しています。

```sh
cargo test
```

```
test result: ok. 3 passed; 0 failed
```

これで準備完了です。
