# ここまでとこの先

## ここまでやったこと

最初のページでは、BLE の地図を頭に入れました。ペリフェラルとセントラルという2つの役割、サービスとキャラクタリスティックという2段のデータ構造、そして Notify（マイコン→スマホ）と Write（スマホ→マイコン）という2方向の操作を押さえたのが出発点です。

準備のページは、このガイドで一番手順が多いところでした。Nordic 製の SoftDevice バイナリをフラッシュの先頭に別途書き込み、プロジェクトの設定ファイルを4つ用意して、アドバタイズするコードを書く。それだけで nRF Connect のスキャンに "micro:bit" が現れました。

気温を送るページでは、チップ内蔵の温度センサを読んで Notify で届くようにしました。CCCD でスマホが購読を宣言して初めて送信が始まること、`select` で BLE イベントの受信と Notify 送信ループを並行して動かしていること、この2点がコードの核心です。

Write でスマホから LED を操作するとき、ハンドラはクロージャの中で動くため `await` を呼べません。そのため受け取った値を `AtomicU8` に保存しておき、別タスクの `display_task` が走査ループの中でそれを読んでパターンを切り替えています。

最後は、nRF Connect の代わりに自分で書いた HTML ファイルで操作できるようにしました。BLE の通信はブラウザとマイコンが直接やりとりするため、通信を中継するサーバーもネイティブアプリも不要です。

この一連の作業を通じて、ファームウェア（Rust）とブラウザ（JavaScript）の両端を自分で書き、サービスとキャラクタリスティックの構造をコードと実機で確かめながら繋ぎ合わせました。

## この先やってみること

届いたか確認する送信 — このガイドで使った Notify はデータを送りっぱなしにします。スマホが届いたことを返事する仕組みも BLE には用意されていて、`#[characteristic]` の `notify` を `indicate` に変えることで切り替えられます。

外から繋ぐセンサ — チップ内蔵の温度センサは精度が粗く、チップ自体の熱も拾います。外付けの温湿度センサや気圧センサを micro:bit に繋ぐと、より正確な値が読めます。センサとマイコンをケーブルで繋いでデータをやりとりする手順が加わりますが、取れた値を Notify で送る部分はこのガイドと同じです。

既製アプリと繋ぐ — スマホには BLE の汎用アプリが多くあります。このガイドでは自分で Web ページを作りましたが、Bluetooth の公開仕様に沿った UUID と値のフォーマットで実装すると、そういったアプリからも読めるようになります。

他のマイコンで動かす — このガイドで使った embassy のタスクモデルは、RP2040（Raspberry Pi Pico）や STM32 系のマイコンでも同じように動きます。チップが変わっても非同期タスクの書き方は共通です。BLE を使う場合は各チップ向けの別のライブラリが必要になります。
