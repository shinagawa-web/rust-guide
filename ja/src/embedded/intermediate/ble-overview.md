# マイコンとスマホが繋がる仕組み

体重計に乗ると、数秒後にアプリに記録される。スマートウォッチは心拍と歩数をスマホに送り続けている。フィットネスバンドが歩数と消費カロリーをスマホに届ける。

こういう「近くにあるデバイスがスマホと繋がる」仕組みを、BLE（Bluetooth Low Energy）と呼びます。

この中級編では、micro:bit をそういうデバイスにします。そのためにまず、BLE がどんな部品で構成されているかを押さえます。

## 2つの役割

BLE の通信では、2つの機器がそれぞれ異なる役割を持ちます。データを持って接続を待つ側を「ペリフェラル」、接続しに行く側を「セントラル」と呼びます。

| 役割 | 担当 | 働き |
|------|------|------|
| ペリフェラル | micro:bit | データを持ち、接続を待つ |
| セントラル | スマホ | 接続しに行き、データを読む |

スマホが micro:bit を見つけて接続しに行きます。この方向は逆になりません。

## データの持ちかた

ペリフェラルが持つデータは、サービスとキャラクタリスティックという2段の構造で整理されています。

- サービスは機能のまとまりです。「気温に関するもの」「LED に関するもの」のような括りです。
- キャラクタリスティックは実際の値です。読み書きの単位になります。

各サービスと各キャラクタリスティックには、それを識別する UUID（固有の番号）が割り当てられています。Bluetooth の仕様で標準的なものが決まっており、コードでは文字列として書きます。

このガイドで作る micro:bit の構成はこうなります。

```text
micro:bit（ペリフェラル）
├─ 気温サービス（UUID: 181a）
│   └─ 気温の値（UUID: 2a6e）  ← Notify でスマホに送る
└─ LED サービス（UUID: 1815）
    └─ LED コマンド（UUID: 2a56）  ← スマホから Write で受け取る
```

```rust
// UUID 181a のサービスを宣言する
#[nrf_softdevice::gatt_service(uuid = "181a")]
struct TemperatureService {
    // UUID 2a6e のキャラクタリスティック。値の型は i16（16ビット整数）
    // notify: micro:bit からスマホへ値を送れる
    // read:   スマホから現在値を読み出せる
    #[characteristic(uuid = "2a6e", read, notify)]
    temperature: i16,
}

// Automation IO（1815）の Digital（2a56）を LED コマンド用に使う
#[nrf_softdevice::gatt_service(uuid = "1815")]
struct LedService {
    // UUID 2a56 のキャラクタリスティック。値の型は u8（8ビット整数）
    // write: スマホから micro:bit へ値を送れる
    #[characteristic(uuid = "2a56", write)]
    command: u8,
}
```

次のページ以降で実際に書くコードはこういう形になります。

`gatt_service` アトリビュートでサービスの UUID を宣言し、`characteristic` アトリビュートでキャラクタリスティックの UUID と操作の種類を指定します。`notify` を付けると micro:bit からスマホへ値を送れるようになり、`write` を付けるとスマホから micro:bit へ値を送れるようになります。

## Notify と Write

キャラクタリスティックの値をやりとりする方法のうち、このガイドでは2つを使います。

```text
  micro:bit                       スマホ
（ペリフェラル）                 （セントラル）
      │                               │
      │  ──── Notify ──────────▶  │  気温を1秒ごとに送りつける
      │                               │
      │  ◀─── Write ────────────  │  LED パターンを送る
```

Notify はペリフェラルからセントラルへの一方的な送信です。身の回りでは：

- スマートウォッチが心拍数をリアルタイムで送る
- 血圧計・体温計が測定値をアプリに送る
- フィットネストラッカーが歩数を定期的に送る

このガイドでは気温センサの値を1秒ごとに送ります。スマホ側は接続後に通知を購読（サブスクライブ）するだけで、あとは更新のたびに自動で届きます。都度ポーリングする必要はありません。

Write はセントラルからペリフェラルへの送信です。身の回りでは：

- スマートロックをスマホで解錠する
- スマート電球の色や明るさをアプリで変える
- デバイスの動作設定（測定間隔・閾値）をアプリから変更する

このガイドではスマホのボタンを押したときに LED の切り替えコマンドを送ります。
