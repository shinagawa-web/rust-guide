# 初級

自分で書いた Rust を、本物のマイコンボードに書き込んで動かします。他の初級編のように既存コードを読んで直すのではなく、実機でできることを1ページずつ増やしていきます。ボードの LED を光らせ、絵を出し、ボタンやセンサに反応させる——読み終えるころには、パソコンの画面の中ではなく、手のひらの上のハードウェアが自分のコードで動いている、という手応えが残ります。

主役は、OS も標準ライブラリも無い地面の上で Rust が動く、という事実です。普段のプログラムが当たり前に使っている `println!` や `Vec` が、ここには無い。その代わり、チップのピンや LED に自分で手が届く。この落差をくぐり抜けるのが、このルートで新しく学ぶところです。

## 用意するもの

このガイドは実機を前提にします。シミュレータではなく、本物のボードに書き込んで、目の前の LED が光るのを確かめます。

- micro:bit v2 — 手のひらサイズのマイコンボードです。5×5 の LED、2つのボタン、加速度センサと温度センサが最初から基板に載っていて、配線もハンダ付けも要りません。書き込み用の回路もボードに載っているので、USB でパソコンにつなぐだけで書き込めます。
- USB ケーブル — micro:bit とパソコンをつなぎます（micro:bit 側は端子の形を確認してください）。

パソコン側のツール（Rust のツールチェーンと書き込みツール）は、2ページ目で一から用意します。

## 題材にするもの

このガイドでは、育てていく1つのプログラムを題材にします。最初は LED を光らせるだけの数行から始めて、ページを進むごとに手を加え、最後には傾きや温度に反応するところまで持っていきます。

できあがっていくコードは、次のリポジトリに置いてあります。各ページはこのコードを少しずつ書き足す形で進みます。

<https://github.com/shinagawa-web/rust-guide-sample-beginner-embedded>

## このガイドの進め方

全 7 ページ。手元にボードを置いて、書き込みながら読み進めてください。

以下の順で進みます。

1. [なぜ Rust で組み込みか](beginner/why-embedded.md) ― 標準ライブラリの無い地面で動く、というこのルートの入口を見る
2. [ボードを光らせるまで](beginner/blink.md) ― ツールを用意し、書き込んで、LED を光らせる
3. [no_std の世界](beginner/no-std.md) ― なぜ `println!` が無いのか、ヒープの無い世界を知る
4. [LED マトリクスに絵を出す](beginner/led-matrix.md) ― 5×5 を組み立てる関数を書き、パソコン上でテストする
5. [ボタンで反応する](beginner/button.md) ― ボタンの入力を読み、押したら絵を変える
6. [センサで動かす](beginner/sensor.md) ― 傾きや温度を読んで、絵を動かす
7. [ここまでとこの先](beginner/wrap-up.md) ― 学んだことを自分の言葉でまとめ、次に何ができるかを見る

2ページ目が最初の山場です。ツールをそろえて最初の書き込みを通すところが、このルートでいちばんつまずきやすく、そして最初の「本物が光った」を味わうところでもあります。
