# ビットとマスクの基本

前のページで、サンプルコードに出てくる Rust の構文を押さえました。ただ `perm.rs` には、パーミッションの整数（`mode`）のビットを直接操作する書き方が出てきて、構文を知っているだけでは読めません。

パーミッションの 10 文字は、OS がファイルごとに持つ 1 個の整数（`mode`）からできています。サンプルコードは、この `mode` から必要なビットを取り出して文字に直しています。ここでは、そのために使う「ビット」「8 進数」「マスク」「シフト」を順に押さえます。

## なぜビット演算で読むのか

パーミッションは、独立した「あり / なし」がたくさん集まったものです。所有者は読めるか、書けるか、実行できるか。グループは、その他は、と続けていくと、全部で 9 個の yes / no があります。

これを 9 個バラバラに持つのは無駄なので、Unix 系 OS は昔から、1 個の整数の中に 1 ビット＝1 つの yes / no として詰め込んできました。`mode` もこの形で渡されます。

OS がビットで持っているものを読むなら、ビットをそのまま扱うのが素直です。必要なビットだけを取り出すマスクと、ビットの位置をそろえるシフトは、そのための定番の道具です。表現とやり方が一致しているので、短く書けて動作も速い。Unix 系のコードがパーミッションをビット演算で扱うのはこのためで、サンプルコードも同じ書き方をしています。

では、この 2 つの道具を使えるように、まず数の表し方から見ていきます。

## 2 進数とビット

コンピュータの中では、整数はビット（0 か 1）の並びとして表されます。Rust では、2 進数のまま数値を書くときに `0b` を先頭に付けます。

```rust
fn main() {
    let n = 0b101;
    println!("{}", n); // 5
}
```

`0b101` は 2 進数の 101 で、10 進数では 5 です。

パーミッションの読み (r)・書き (w)・実行 (x) は、それぞれ 1 ビットで表せます。権限があれば 1、なければ 0 です。3 つ並べると 3 ビットになります。

```text
rwx = 0b111 = 7   (全部あり)
r-x = 0b101 = 5   (書きだけなし)
r-- = 0b100 = 4   (読みだけ)
```

## 8 進数とパーミッション

ビット 3 つは、ちょうど 8 進数 1 桁で表せます（8 は 2 の 3 乗だからです）。Rust では 8 進数を `0o` を先頭に付けて書きます。

パーミッションの 9 ビットは、所有者・グループ・その他の 3 ビットずつに分かれていました。なので、8 進数で書くと 3 桁にきれいに対応します。

```text
rwx r-x r-x
111 101 101
 7   5   5   →  0o755
```

`ls -l` の `rwxr-xr-x` が `chmod 755` の 755 と同じものを指しているのは、このためです。

サンプルのテストに出てくる `0o100755` は、下 3 桁の `755` がこのパーミッション部分です。上の桁（`100`）はファイルの種別を表すビットで、これはバグを直すページで扱います。

## マスク ― 必要なビットだけ取り出す

`mode` には、種別と、所有者・グループ・その他の権限が、すべて 1 個の整数の中にまとめて詰まっています。一方で `rwxr-xr-x` は、この中から「その他の 3 ビット」というように一部分だけを取り出し、1 つずつ調べて組み立てます。まとめて入っている `mode` から、目的のビットだけを切り出す必要があるわけです。

整数から特定のビットだけを取り出すには、AND 演算 `&` を使います。

`&` は 2 つの数をビットごとに見て、両方が 1 のときだけ 1 にします。取り出したいビットだけを 1 にした数（マスク）と AND を取ると、それ以外のビットはすべて 0 になります。

```text
  0b111_101_101   (0o755)
& 0b000_000_111   (0b111 = 下位 3 ビットだけ 1)
-----------------
  0b000_000_101   (= 0b101 = その他の r-x)
```

`& 0b111` は「下位 3 ビットだけ残して、ほかは 0 にする」という意味になります。これでその他のユーザーの権限が取り出せます。

## シフト ― ビットの位置をずらす

`& 0b111` で取り出せるのは下位 3 ビット、つまりその他のユーザーの権限だけです。同じマスクで所有者やグループの権限も取り出したいのですが、これらのビットは上の方にあるので、`& 0b111` ではすくい取れません。

そこで、取り出したい 3 ビットを先に下位へずらしてからマスクします。ビットを右へずらすには、右シフト `>>` を使います。`>>` はビットの並びを指定した数だけ右へずらし、はみ出したビットは捨てます。

```text
  0b111_101_101   (0o755)
>> 6
-----------------
  0b000_000_111   (= 0b111 = 所有者の rwx)
```

所有者のビットは 6 つ左にあるので、`>> 6` で下位に下ろせます。グループは 3 つ左なので `>> 3` です。

## 組み合わせて読む

マスクとシフトを組み合わせると、9 ビットから 3 ビットずつ取り出せます。サンプルコードはこう書いています。

```rust
rwx((mode >> 6) & 0b111); // 所有者
rwx((mode >> 3) & 0b111); // グループ
rwx(mode & 0b111);        // その他
```

まずシフトで対象の 3 ビットを下位に下ろし、`& 0b111` でその 3 ビットだけを残しています。

取り出した 3 ビットの中身は、さらに 1 ビットずつ調べます。読みは上から 1 ビット目、書きは 2 ビット目、実行は 3 ビット目です。それぞれ対応するマスクと AND を取り、0 でなければ権限あり、と判定します。

```rust
let r = if bits & 0b100 != 0 { 'r' } else { '-' };
let w = if bits & 0b010 != 0 { 'w' } else { '-' };
let x = if bits & 0b001 != 0 { 'x' } else { '-' };
```

`bits & 0b100` は読みのビットだけを取り出す式で、その結果が 0 でなければ読み権限がある、という意味です。書き・実行も同じ要領です。

これで、整数から 9 ビット分の `rwxr-xr-x` を組み立てる仕組みが読めるようになりました。残るは先頭の種別 1 文字ですが、それを読む前に、そもそもこの整数 `mode` を OS からどう受け取っているのかを知っておく必要があります。次のページでは、ファイルのメタデータの読み取りを見ていきます。
