# 変更をふり返る

ディレクトリを渡しても先頭が `d` になり、テストもそれを守るようになりました。ここまでで手を入れたのは `perm.rs` の 2 か所だけです。最後に、自分が何をどう変えたのかを見返し、自分の言葉でまとめます。変更を自分の言葉で言えるようになることが、この初級編のねらいです。

## 変更を見返す

まず、自分の変更を `git diff` で見返します。`git diff` は、いま手元で変えた内容を、変えた行だけ並べて見せてくれます。

```sh
$ git diff
```

```diff
--- a/src/perm.rs
+++ b/src/perm.rs
@@ -11,8 +11,12 @@ pub fn format_mode(mode: u32) -> String {
 }
 
 // 先頭 1 文字の種別を返す。
-fn file_type(_mode: u32) -> char {
-    '-'
+fn file_type(mode: u32) -> char {
+    if mode & 0o170000 == 0o040000 {
+        'd'
+    } else {
+        '-'
+    }
 }
 
 // 3 ビットを rwx の 3 文字に変換する。
@@ -32,4 +36,10 @@ mod tests {
         // 0o100755: 通常ファイルで rwxr-xr-x
         assert_eq!(format_mode(0o100755), "-rwxr-xr-x");
     }
+
+    #[test]
+    fn formats_directory() {
+        // 0o040755: ディレクトリで rwxr-xr-x
+        assert_eq!(format_mode(0o040755), "drwxr-xr-x");
+    }
 }
```

`-` で始まる行が消したもの、`+` で始まる行が足したものです。変えたのは 2 か所——`file_type` の中身と、末尾に足したディレクトリのテスト——だけだと確かめられます。

## 何をどう変えたか、文章にする

diff を見ながら、変更を自分の言葉で説明してみます。次の 4 つが書ければ十分です。

- 直したもの: ディレクトリを渡すと、先頭が `-` のままになっていた。
- 原因: `file_type` が `mode` を見ずに、いつも `-` を返していた。
- 直し方: `mode & 0o170000` で種別のビットを取り出し、ディレクトリの値 `0o040000` なら `d`、それ以外は `-` を返すようにした。
- 確認: 既存のテストは通ったまま、ディレクトリのテストを足して、あとで元に戻っても気づけるようにした。

コードは読めば追えますが、「なぜその変更で直るのか」を言葉にできるかどうかは別の力です。ここまで一段ずつ下りてきたぶん、原因も直し方もビットのレベルで説明できるはずです。

## ふり返り

ここまでで、`ls -l` の 10 文字が OS の持つ 1 個の整数からできていることを知り、その整数を `main.rs` と `perm.rs` がどう読んで文字に直しているかを読み、ビットを手がかりに `file_type` を直し、テストで守り、変更を自分の言葉でまとめました。

やったのは小さな 1 か所の修正ですが、その 1 か所にたどり着くために、文字の下にある整数、整数の下にあるビット、という低レイヤの層を一段ずつ下りてきました。既存のコードを読んで、必要なところだけ直せる——この進め方は、題材がパーミッションでなくなっても同じように使えます。
