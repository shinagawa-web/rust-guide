# パーミッションの数字を読む

前のページで、サンプルコードに出てくる Rust の構文を押さえました。残るは、`perm.rs` が `mode` という整数をどうやって `rwxr-xr-x` に直しているか、です。これは構文の知識だけでは読めないので、ここで仕組みを見ておきます。

鍵になるのは、`mode` を 8 進数で見ることです。

## パーミッションは 8 進数の数字

`mode` を 8 進数で書くと、パーミッションの部分は 3 桁の数字になります。`ls -l` の `rwxr-xr-x` は、`chmod 755` の `755` と同じものです。

```text
rwxr-xr-x  =  755
```

3 桁は、左から所有者・グループ・その他に対応します。`755` なら所有者が 7、グループが 5、その他が 5 です。

3 桁にきれいに分かれるのは、1 人分の権限（読み・書き・実行の組み合わせ）がちょうど 0〜7 の 8 通りで、8 進数 1 桁で表せるからです。

## 各桁の読み方

1 桁の数字は、読み・書き・実行に次の値を割り当てて、その合計になっています。

```text
読み (r) = 4
書き (w) = 2
実行 (x) = 1
```

たとえば `7` は 4 + 2 + 1 で読み書き実行のすべて、`5` は 4 + 1 で読みと実行、という具合です。0〜7 をすべて並べると次のようになります。

| 数字 | 権限 | 意味 |
|---|---|---|
| 0 | `---` | なし |
| 1 | `--x` | 実行のみ |
| 2 | `-w-` | 書きのみ |
| 3 | `-wx` | 書き・実行 |
| 4 | `r--` | 読みのみ |
| 5 | `r-x` | 読み・実行 |
| 6 | `rw-` | 読み・書き |
| 7 | `rwx` | 読み・書き・実行 |

`755` は、この表で `rwx` `r-x` `r-x` となり、つなげると `rwxr-xr-x` になります。

## 数字を取り出す

サンプルコードは、`mode` から 3 つの桁を取り出して、この表で文字に直しています。桁を取り出すのは、割り算と余りです。

```rust
let owner = (mode / 64) % 8; // 下から 3 桁目
let group = (mode / 8) % 8;  // 下から 2 桁目
let other = mode % 8;        // 下から 1 桁目
```

8 進数の 1 桁は 0〜7 なので、`% 8`（8 で割った余り）で一番下の 1 桁が取り出せます。1 つ上の桁は 8 で割って下ろしてから、さらに 1 つ上は 64（8 × 8）で割って下ろしてから、余りを取ります。

取り出した数字を、先ほどの表（コードでは `RWX`）で文字に直して並べます。

```rust
s.push_str(RWX[owner as usize]);
s.push_str(RWX[group as usize]);
s.push_str(RWX[other as usize]);
```

これで `rwxr-xr-x` の 9 文字が組み立てられます。

## 先頭の種別

残るのは先頭の 1 文字です。`mode` を 8 進数で見ると、権限の `755` の上にもう数桁あります。テストに出てきた `0o100755` の、上の `100` の部分です。この上の桁が、ファイルの種別（通常ファイルかディレクトリか）を表しています。

サンプルの `file_type` は、この桁を見ずにいつも `-` を返しています。これがバグの正体ですが、直すのはもう少し先のページです。

ここまでで、`mode` という整数から 10 文字を組み立てる仕組みがわかりました。次のページでは、そもそもこの `mode` を OS からどう受け取っているのか――ファイルのメタデータの読み取りを見ていきます。
