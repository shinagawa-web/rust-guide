# file_type を書き直す

前のページで、`file_type` が `mode` を見ずにいつも `-` を返していることを読みました。ここでは `mode` から種別を読み取り、ディレクトリなら `d`、それ以外なら `-` を返すように書き直します。

## 種別はどのビットか

パーミッションの 9 ビットは `mode` の下位にありました。種別は、そのさらに上の桁に入っています。「ビットとマスクの基本」で見た `0o100755` を思い出すと、下 3 桁の `755` が権限で、上の `0o100000` が「通常ファイル」を表す種別でした。

種別には、Unix で決まった値が割り当てられています。今回必要なのは 2 つです。

```text
通常ファイル  0o100000
ディレクトリ  0o040000
```

この `0o100000` や `0o040000` も、「ビットとマスクの基本」で見た 8 進数（`0o`）の書き方です。`0o755` と同じく、それぞれ 1 個の整数を表しています。

`mode` の中でこの種別が入っている桁だけを取り出して、ディレクトリの値と一致するかを調べれば、種別が判定できます。

## 種別の桁を取り出す

権限を取り出したときと同じように、マスクで種別の桁だけを残します。種別が占めるのは `0o170000` の位置です。

```text
  mode
& 0o170000   (種別の桁だけ 1)
```

`mode & 0o170000` とすると、権限などほかのビットが 0 になり、種別の値だけが残ります。通常ファイルなら `0o100000`、ディレクトリなら `0o040000` が得られます。

## d か - かを返す

取り出した種別がディレクトリの値と一致すれば `d`、そうでなければ `-` を返します。

```rust
// 先頭 1 文字の種別を返す。
fn file_type(mode: u32) -> char {
    if mode & 0o170000 == 0o040000 {
        'd'
    } else {
        '-'
    }
}
```

引数を `_mode` から `mode` に変えているのは、今度は中で使うからです。`mode & 0o170000` で種別を取り出し、それが `0o040000`（ディレクトリ）なら `d`、それ以外は今までどおり `-` を返します。

> 📝 種別はディレクトリと通常ファイルだけではありません。たとえばシンボリックリンクなら、`ls -l` の先頭は `l` になります。ただ今回直したいのはディレクトリが `-` になる点なので、ここではディレクトリだけを見分け、それ以外はまとめて `-` にしています。種別をひととおり見分けるところまでは、このガイドでは踏み込みません。

## 動かして確かめる

書き直したら、最初にうまくいかなかったディレクトリで試します。

```sh
$ cargo run -- src
```

実行結果

```text
drwxr-xr-x  src
```

先頭が `d` になりました。通常ファイルも試すと、こちらは今までどおりです。

```sh
$ cargo run -- sample/script.sh
```

実行結果

```text
-rwxr-xr-x  sample/script.sh
```

種別を見て先頭の 1 文字を決められるようになりました。次のページでは、この直した動きが元に戻らないよう、ディレクトリのテストを追加します。
