# 所有権

Rust を書きはじめて多くの人が最初に戸惑うのが所有権です。ついさっきまで使えていた変数が、ある行を境にコンパイラから「その値はもう使えない」と止められる。他の言語では見なかった規則で、最初は理不尽にすら感じます。

けれどこれは、Rust が一つの問題を、実行中のコストなしに解くために選んだ仕組みです。その問題とは、プログラムが使ったメモリを、誰がいつ片付けるのか、というものです。所有権が腑に落ちれば、あの「もう使えない」も、ライフタイムや借用のエラーも、同じ一つの理屈の続きとして読めるようになります。

## この節の歩き方

所有権は、この一つの動機からひとつながりに導けます。順番に積み上がるように並べているので、上から読むと迷いません。

- スタックとヒープ — そもそも「片付ける」が問題になるのはメモリのどこか
- 誰がいつ片付けるか — 既存の言語の答えとその弱点、そして両方を避ける Rust の答え＝所有権
- move — 値を渡すと所有権が移り、渡した元が使えなくなる理由
- 借用 — 所有権を渡さずに値を貸し借りする `&` と `&mut`

一本の道として読めるようにしていますが、気になるところから拾い読みしてもかまいません。所有権が腑に落ちれば、ライフタイムや借用のエラーも同じ理屈の続きとして読めるようになります。
