# C言語 から Rust へ

C言語 を書いてきた人に向けた入口です。Rust の基礎を一から並べるのではなく、C と同じ感覚で書けるところは一気に流し、そうはいかないところにページを割きます。すでに知っている C を土台にして、Rust で違うところだけを覚えていく、という進め方です。

## この構成について

C 経験者にとって、Rust の見た目の多くは馴染みのあるものです。変数、条件分岐やループ、関数、struct、配列。C 系の構文なので、書き方の綴りこそ違え、大半はそのまま読めます。ここを一から丁寧に説明されても、退屈なだけで得るものは多くありません。

一方で、C と同じ感覚では書けないところがいくつもあります。あなたはすでに `malloc` と `free` で「誰がいつ解放するか」を手で管理し、ポインタで値を読み書きしてきました。Rust が変えるのは、そこです。

- 手でやっていたメモリ管理の規律を、コンパイラが所有権と借用で肩代わりして強制する。use-after-free も二重解放も、コンパイルの時点で弾かれる
- ヌルポインタの代わりに、`Option` で「無いかもしれない」を型に出す
- 返り値のエラーコードや `errno` の代わりに、`Result` と `?` で、確認し忘れができない形にする
- `void*` と関数ポインタで手作りしていた汎用を、trait とジェネリクスで型安全に書く
- タグを自分で管理する `union` の代わりに、タグ付きのデータを持つ enum を使い、`match` で取り違えを防ぐ
- C につきものの未定義動作を、定義された挙動と、必要なときだけ開ける `unsafe` に置き換える

ここが、C から来た人がつまずきやすく、同時に Rust を Rust たらしめているところです。

だからこのセクションは、二段構えにします。

- 前半で、C とほぼそのまま通じるものを一気に確認します。「C のこれは Rust のこれ」という対応で流し、既定の違いだけを拾います。
- 後半で、C と同じ感覚では書けないところを、一つずつじっくり見ます。「C ではこう、Rust ではこう、なぜそうなっているのか」を軸に、C↔Rust のコードを並べて進めます。

後半の先頭には所有権を置きます。借用もスライスも文字列も、値を誰が持ちいつ解放するかという所有権の上に積み上がるので、ここを最初に固めておくと、あとが楽になります。
