# 言語別ガイド

Rust の解説書はたくさんありますが、どれも Rust を一から説明するものです。あなたがすでに持っている知識と Rust がどう対応するかは、書いていません。

Go を 3 年書いてきたなら、変数・関数・構造体・エラーの返し方はすでに知っています。知らないのは「GC が無い世界でメモリをどう管理するか」だけです。それを一から教わるのは遠回りです。

言語別ガイドは、あなたがすでに知っている言語を土台に、Rust で違うところだけを学ぶための入口です。馴染みのある概念は素早く確認し、Rust ならではのところに時間を使います。Rust の新しい概念のほとんどは、あなたが知っている言語の何かと対応しています。「Go の interface の代わりに trait」「C の void* の代わりに trait」という形で示されると、ゼロから覚えるよりも定着が早くなります。

## 言語ごとに、つまずく場所が違う

「Rust が難しい」と感じる場所は、出身言語によって変わります。

C から来た人にとっての壁は所有権です。ポインタとメモリ管理はすでに肌感覚で知っている。ただそれを手でやっていたものが、コンパイラに移ります。

Go から来た人も所有権がメインの壁ですが、角度が違います。Go では GC が面倒を見てくれていたものを、Rust はコンパイル時に静的に決める。同じ「メモリの安全性」でも、手段がまるで違います。

TypeScript から来た人は壁の数が多い。GC が消えて、null が消えて、throw が消えます。一度に変わるものが多いぶん、最初は面食らいます。

- [C言語 から Rust へ](from-c.md) — ポインタと手動メモリ管理が、所有権と借用に変わる
- [Go から Rust へ](from-go.md) — GC の代わりに、コンパイル時にメモリを管理する
- [TypeScript から Rust へ](from-typescript.md) — GC が消えて、null が消えて、throw が消える

## 読んだ後の動線

このガイドは対応関係を掴む入口です。Rust 自身の論理で概念を深く理解したいなら、[言語の仕組み](../concepts/)で補えます。所有権・ライフタイム・トレイトなどを Rust の側から積み上げる解説があります。

次にどこへ進むかは、何を作りたいかで変わります。同じ言語の出身でも、CLI ツールを書きたい人と Web API を書きたい人では必要な知識が違います。[Web バックエンド](../web-backend/)・[CLI ツール](../cli/)・[WebAssembly](../wasm/)・[低レイヤ](../low-level/)・[組み込み](../embedded/)といった目的別のガイドがあるので、作りたいものが見えてきたら参考にしてください。
