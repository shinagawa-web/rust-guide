# Go から Rust へ

Go を書いてきた人に向けた入口です。Rust の基礎を一から並べるのではなく、Go と同じ感覚で書けるところは一気に流し、そうはいかないところにページを割きます。すでに知っている Go を土台にして、Rust で違うところだけを覚えていく、という進め方です。

## この構成について

Go 経験者にとって、Rust の多くは実は馴染みのあるものです。変数、条件分岐やループ、関数、構造体、配列やマップ。書き方こそ違え、考え方は Go とほとんど同じです。ここを一から丁寧に説明されても、退屈なだけで得るものは多くありません。

一方で、Go と同じ感覚では書けないところがいくつかあります。

- GC が無く、所有権でメモリを管理する
- nil が無く、Option で「無いかもしれない」を表す
- エラーを値で返すのは Go と同じだが、Rust では Result という型と `?` で、確認し忘れができない形に徹底される
- 共通の振る舞いを、interface ではなく trait で表す

ここが、Go から来た人がつまずきやすく、同時に Rust を Rust たらしめているところです。

だからこのセクションは、二段構えにします。

- 前半で、Go とほぼそのまま通じるものを一気に確認します。「Go のこれは Rust のこれ」という対応で流し、細かい違いだけを拾います。
- 後半で、Go と同じ感覚では書けないところを、一つずつじっくり見ます。「Go ではこう、Rust ではこう、なぜそうなっているのか」を軸に、つまずきやすい順で並べます。

後半の先頭には所有権を置きます。借用もエラー処理も trait も、その上に乗っているので、ここを最初に固めておくと、あとが速くなります。
