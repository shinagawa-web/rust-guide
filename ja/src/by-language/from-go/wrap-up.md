# Go から Rust へ、ここまで

[Go とほぼ同じところ](basics.md)で、変数・制御フロー・関数・構造体・Vec / HashMap を一気に流しました。書き方は違っても、考え方は Go とほとんど同じでした。そのあと、同じ感覚では書けなかった数か所を、一つずつ見てきました。

振り返ると、Rust の多くは Go の知識の上にそのまま乗り、引っかかるのは次の数か所だけでした。

- [所有権と借用](ownership.md) — GC の代わりに、コンパイル時にメモリを管理する
- [nil の代わりに Option](option.md) — 「無いかもしれない」を型で表す
- [エラーの扱い](result.md) — Go の `(値, err)` から、Result と `?` へ
- [interface の代わりに trait](trait.md) — 共通の振る舞いを表す
- [データを持つ enum と match](enum.md) — 種類ごとに違うデータを型で分け、扱い漏れを防ぐ
- [並行性](concurrency.md) — 共有データの競合をコンパイル時に止める

これらは別々の機能に見えて、根はつながっています。中心にあるのは所有権です。値を誰が持ち、いつ手放すか。この一点が、メモリの管理でも、エラーや「無い」の扱いでも、複数スレッドの共有でも、同じ仕組みになって効いてきます。Rust がうるさく見える場面の多くは、この所有権が「今それは安全でない」と教えているだけです。

Go で身につけた地図は、そのまま使えます。その上に所有権という一点を重ねれば、Rust はもう遠い言語ではありません。
