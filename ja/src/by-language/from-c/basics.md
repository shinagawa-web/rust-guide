# ほぼ読めるが、既定が違う

C言語 を書いてきたなら、Rust のコードは見た目の大半がすぐ読めます。変数を宣言して、条件で分岐し、ループを回し、関数を呼び、struct にデータをまとめる。C 系の構文なので、綴りこそ違え、考え方はほとんど同じです。

ただ、いくつか既定が違います。しかもその違いは、たいてい C で事故になりやすかったところを塞ぐ方向に振ってあります。このページは、その「ほぼそのまま通じる」部分を対応で流しながら、既定がずれるところだけを拾います。「C のこれは Rust のこれ」と読み替えられれば十分なので、流し読みでかまいません。同じ感覚では書けないところは、次の章から一つずつ見ます。

## 変数 — 既定が不変で、未初期化が無い

C の変数は、書き換えられるのが既定でした。Rust は逆で、`let` で束ねた変数は書き換えられないのが既定です。書き換えたいときだけ `mut` を付けます。

```c
// C
int x = 10;
int y = 20;
y = 25;
```

```rust
// Rust
# fn main() {
let x = 10;
let mut y = 20;
y = 25;
# println!("{x} {y}");
# }
```

型は基本的に推論されます。明示するときは変数のうしろに書きます（`let y: i32 = 20;`）。C の `int y = 20;` と型の位置が逆になる点だけ、目が慣れるまでの違和感です。

もう一つの違いは初期化です。C では `int x;` と宣言だけしておいて、あとから代入できました。代入する前に読むと、その変数にはメモリのその場所に前から残っていた値がそのまま出ます。何が入っているかは決まっておらず、実行のたびに変わることもあって、これがバグの温床でした。Rust は、初期化していない変数を読むコードはコンパイルを通しません。

```c
// C
int x;
printf("%d\n", x); // その場所に残っていた値が出る（未定義動作）
```

```rust
// Rust
fn main() {
    let x: i32;
    println!("{x}"); // コンパイルエラー：x が初期化されていない
}
```

宣言だけ先に書くこと自体は Rust でもできますが、読む前に必ずどこかで代入されている、とコンパイラが確かめられる形でなければ通りません。「未初期化のまま読む」が、そもそも書けなくなっています。

## 型の変換 — 暗黙変換が無い

C は、型の違う値を代入や演算で暗黙に変換しました。広い型から狭い型へ入れても、符号付きと符号なしをまたいでも、警告なく通ります。何が起きるかをコードで見ると、こうです。

```c
// C
long big = 3000000000;
int n = big;        // int に収まらず -1294967296 に化ける

int neg = -1;
unsigned u = neg;   // 符号なしに読み替えられ 4294967295 になる

double x = 3.9;
int t = x;          // 小数が切り捨てられ 3 になる
```

どれも代入の一行で静かに起きます。Rust は、型が違えば暗黙には変換しません。`as` を付けずに違う型へ入れようとすると、その時点でコンパイルエラーになります。

```rust
// Rust
fn main() {
    let big: i64 = 3_000_000_000;
    let n: i32 = big; // コンパイルエラー：i64 を i32 に暗黙には入れられない
}
```

変換したいなら `as` を書きます。その一行が残るので、桁が落ちる変換だと目で分かります。

```rust
// Rust
# fn main() {
let big: i64 = 3_000_000_000;
let n = big as i32;   // 切り詰めると明示（-1294967296）

let neg: i32 = -1;
let u = neg as u32;   // 符号なしへの読み替えも明示（4294967295）

let x = 3.9_f64;
let t = x as i32;     // 小数の切り捨ても明示（3）
# println!("{n} {u} {t}");
# }
```

`i32` を `i64` に入れるような「明らかに広い方へ移すだけ」でも、Rust は暗黙変換しません。どこで型が変わったかがコードに必ず残るので、上のような桁落ちや符号の取り違えを目で追えます。

## 整数のあふれ — 静かに巻き戻らない

あふれの扱いも違います。C では、符号なし整数のあふれは静かに巻き戻り、符号付きのあふれは未定義動作でした。

```c
// C
unsigned char c = 255;
c = c + 1; // 静かに 0 へ巻き戻る
```

Rust は、あふれを静かに見逃しません。あふれると分かる値は、コンパイルの時点で弾かれます。

```rust
// Rust
fn main() {
    let c: u8 = 255;
    let d = c + 1; // コンパイルエラー：255 + 1 は u8 をあふれるとコンパイラが見抜く
}
```

実行時に決まる値なら、デバッグビルドではあふれた瞬間にパニックして気づけます。巻き戻りをそもそも意図しているなら、その意図を `wrapping_add` という名前で選びます。

```rust
// Rust
# fn main() {
let c: u8 = 255;
let d = c.wrapping_add(1); // 巻き戻したいと明示すれば 0
# println!("{d}");
# }
```

## 制御フロー — if / for / match

`if` は C とほぼ同じ感覚で書けます。違いは、条件を丸括弧で囲まないことと、本体の波括弧を省略できないことです。波括弧が必須なので、C で `else` がどちらの `if` に付くかで悩む、あの取り違えがそもそも生じません。

```c
// C
if (n > 0) {
    printf("positive\n");
} else {
    printf("zero or negative\n");
}
```

```rust
// Rust
# fn main() {
# let n = 1;
if n > 0 {
    println!("positive");
} else {
    println!("zero or negative");
}
# }
```

`for` は範囲やイテレータを回す形が基本です。C の `for (i = 0; i < 5; i++)` のようなカウンタ式のループはありません。範囲は `0..5` で表します（`5` は含みません）。

```c
// C
for (int i = 0; i < 5; i++) {
    printf("%d\n", i);
}
```

```rust
// Rust
# fn main() {
for i in 0..5 {
    println!("{i}");
}
# }
```

`if` は値を返す式でもあります。C の三項演算子 `cond ? 1 : 2` にあたる書き方を、`if` そのものでできます。

```c
// C
int n = cond ? 1 : 2;
```

```rust
// Rust
# fn main() {
# let cond = true;
let n = if cond { 1 } else { 2 };
# println!("{n}");
# }
```

C の `switch` にあたるのが `match` です。ここは既定が二つ違います。

一つは `break` の要不要です。C の `switch` は、`break` を書き忘れると、その `case` の処理に続けて次の `case` の処理までそのまま実行されました。フォールスルーと呼ばれる挙動です。下は `n` が `1` のときでも、`case 1` から `case 2` へ実行が流れ込み、`onetwo` と出ます。

```c
// C（n == 1）
switch (n) {
    case 1:
        printf("one");   // break が無い
    case 2:
        printf("two");   // case 1 から実行が流れ込み、ここも動く
        break;
    default:
        printf("other");
}
// 出力: onetwo
```

`match` にフォールスルーはありません。一致した分岐だけが実行され、`break` も要りません。同じ `n == 1` でも、出るのは `one` だけです。

```rust
// Rust
# fn main() {
# let n = 1;
match n {
    1 => print!("one"),
    2 => print!("two"),
    _ => print!("other"),
}
# }
```

もう一つは網羅性です。`match` は、取りうる値をすべて扱ったかをコンパイラが確かめます。C の `default` にあたるのが `_`（それ以外すべて）で、これを落として扱わない値が残ると、コンパイルエラーになります。ここでは値で分岐できる程度に覚えておけば十分で、`match` の本領（データを持つ enum の網羅）はあとの章で見ます。

## 関数 — fn

C の `int add(int a, int b)` のような宣言は、Rust では `fn add(a: i32, b: i32) -> i32` になります。Rust は戻り値の型を `->` のうしろに書く点だけが、目新しいところです。

一つ違うのが戻り値の返し方です。C は `return` で返しますが、Rust は関数の最後の式がそのまま戻り値になるので、`return` を省けます。下の `add` は、末尾に `;` を付けない式 `a + b` だけで返しています（Rust でも `return a + b;` と書けます）。

```c
// C
int add(int a, int b) {
    return a + b;
}
```

```rust
// Rust
# fn main() {
#     println!("{}", add(2, 3));
# }
fn add(a: i32, b: i32) -> i32 {
    a + b // 末尾の式が戻り値。return は不要
}
```

宣言と定義を分ける必要が無いのも C との違いです。C ではヘッダにプロトタイプを書き、`.c` に本体を書いて、順番や重複に気を配りました。Rust は定義が一つあれば足り、呼ぶ側より後に定義してもかまいません。

## struct とメソッド — struct と impl

struct の定義は C とよく似ています。違いは、その struct に付く操作の置き場所です。C では struct を第一引数に取る関数を別に並べましたが、Rust はその struct 専用の `impl` ブロックにメソッドとしてまとめます。

```c
// C
struct Point {
    int x;
    int y;
};

struct Point point_add(struct Point p, struct Point q) {
    struct Point r = { p.x + q.x, p.y + q.y };
    return r;
}
```

```rust
// Rust
# fn main() {
#     let p = Point { x: 1, y: 2 };
#     let q = Point { x: 3, y: 4 };
#     let _ = p.add(&q);
# }
struct Point {
    x: i32,
    y: i32,
}

impl Point {
    fn add(&self, q: &Point) -> Point {
        Point { x: self.x + q.x, y: self.y + q.y }
    }
}
```

C で `point_add(p, q)` と呼んでいたものが、Rust では `p.add(&q)` になります。第一引数に渡していた「操作する当のもの」が、`self` として受け取り側に移った、と考えると読めます。この `self` に付いた `&` が所有権と借用の話で、次の章の本題です。

## 配列と動的配列 — 配列と Vec

要素数がコンパイル時に決まる配列は、C の配列とほぼ同じです。書き方は `[T; N]` で、要素の型と個数を書きます。

```c
// C
int xs[3] = {1, 2, 3};
```

```rust
// Rust
# fn main() {
let xs: [i32; 3] = [1, 2, 3];
# println!("{xs:?}");
# }
```

要素数が実行時に増える動的な配列は、C では手作業でした。確保した領域へのポインタとは別に、今いくつ入っているか（長さ）といくつ分確保したか（容量）を自分で変数に持ち、満杯になれば `realloc` で取り直して容量を更新します。要素を一つ足すだけでも、この一式を自分で回します。

```c
// C — 満杯の配列に要素を1つ足す
size_t cap = 3;   // 確保してある個数
size_t len = 3;   // 使っている個数
int *xs = malloc(cap * sizeof(int));

if (len == cap) {                        // 満杯になったら自分で拡張
    cap *= 2;
    xs = realloc(xs, cap * sizeof(int));
}
xs[len] = 4;
len += 1;
// 使い終わったら free(xs);
```

Rust でこれにあたるのが `Vec` です。長さも容量も、再確保も、使い終わったあとの解放も `Vec` 自身が持つので、足すのは `push` 一つで済みます。

```rust
// Rust
# fn main() {
let mut xs = vec![1, 2, 3];
xs.push(4);
println!("{xs:?}"); // [1, 2, 3, 4]
# }
```

出力の `{xs:?}` の `:?` は、`Vec` のような中身のある値を見やすく出すための指定です。

---

ここまでは、知っている C がほぼそのまま使えました。既定がいくつか不変や明示の側へ振ってあるだけで、読み書きの勘はそのまま通じます。次の章からが本題です。C と同じ感覚では書けないところ、その土台になる所有権に入ります。手で `malloc` と `free` をしていたところを、Rust がどう引き受けるのかを見ます。
