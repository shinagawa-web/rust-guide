# C言語 から Rust へ、ここまで

[ほぼ読めるが、既定が違う](basics.md)で、不変がデフォルト・未初期化がない・暗黙の型変換がない・整数のあふれといった、書き方の違いをひととおり確認しました。C 系の構文なので読むだけなら難しくなく、違いはわずかでした。そのあと、同じ感覚では書けない数か所を一つずつ見てきました。

振り返ると、C でプログラマが守っていたことが、Rust では型とコンパイラの側に移っていました。

- [ポインタと所有権](ownership.md) — `malloc`/`free` の手動管理を、コンパイル時のメモリ管理に
- [配列の借用](borrowing.md) — ポインタと長さの別管理を、スライスにまとめて
- [NULL の代わりに Option](option.md) — ヌルポインタを型で締め出す
- [文字列](strings.md) — NUL 終端と `char*` から、`&str` と `String` と UTF-8 保証に
- [エラー処理](error.md) — 戻り値の確認忘れを、`Result` と `?` で防ぐ
- [union の代わりに enum と match](enum.md) — タグの手管理を、タグ付き enum と網羅チェックに
- [void* の代わりに trait](trait.md) — 関数ポインタと `void*` の汎用を、trait で型安全に
- [未定義動作と unsafe・FFI](unsafe.md) — 未定義動作を定義された挙動に、逃げ道は `unsafe` で明示
- [並行性](concurrency.md) — mutex のかけ忘れを、`Mutex<T>` の構造で防ぐ
- [Makefile の代わりに Cargo](cargo.md) — ビルドと外部ライブラリ管理を一本に

これらは別々の機能に見えて、根はつながっています。所有権は、メモリの解放でも、借用の期間でも、スレッド間の共有でも、同じ仕組みとして効いてきます。C で「誰がいつ解放するか」をコードに書いて管理していたところを、Rust はコンパイラが管理します。コンパイラに弾かれる場面の多くは、その所有権が「今それは安全でない」と教えているだけです。

C で身につけた感覚——ポインタが何を指しているか、メモリがどこにあるか、バイトがどう並んでいるか——はそのまま Rust で使えます。その上に所有権という一点を重ねると、Rust はもう遠い言語ではありません。
