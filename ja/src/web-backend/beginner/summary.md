# まとめ

このガイドでは、Rust で書かれた Web サーバーの issue を一つ解きました。

## 今回学んだこと

- `cargo run` / `cargo test` でプロジェクトを動かす
- `let` / `mut` / `fn` / `if` / `for` の基本的な書き方
- `mod` / `use` / `pub` でファイルをまたいでコードを読む
- `struct` でデータをひとまとめにする
- `Option` で値がない場合を扱う
- `Result` で成功・失敗を型で表す
- コンパイルエラーを読んで修正箇所を特定する
- テストの期待値を変更内容に合わせて更新する

## 今回スルーしたこと

コード中に出てきたものの、説明を省いたものがいくつかあります。

- `async` / `await` — 非同期処理の仕組み
- `&` と `&mut` — [借用の詳細](../../concepts/ownership/borrow.md)
- [`String` と `&str` の違い](../../concepts/ownership/borrow.md)
- トレイト・ジェネリクス・クロージャ・イテレータ

借用と文字列の扱いは上のリンク先（所有権）で読めます。残りは中級以降のガイドで扱います。

## 最後に

このガイドの目的は、Rust がわからない状態で現場に入ったとき最初の 1 週間を乗り切ることでした。それができたなら、このガイドを作った甲斐があります。Rust は難しいと言われますが、全部知らなくても現場はなんとかなります。これからのお仕事、頑張ってください。
